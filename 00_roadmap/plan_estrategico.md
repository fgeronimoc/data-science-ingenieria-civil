# Plan Estratégico — Data Science en Ingeniería Civil

## Objetivo de largo plazo

Construir y consolidar una nueva área dentro del sector construcción peruano que integre Data Science, análisis geoespacial y visión computacional para mejorar la toma de decisiones en proyectos de infraestructura (puentes, carreteras, obras civiles). Independientemente del contexto laboral actual, esta base es un activo profesional permanente.

---

## Pilares del proyecto

### Pilar 1 — Riesgo climático aplicado a obra
Cruzar datos meteorológicos (SENAMHI, ERA5) con cronogramas de construcción para predecir ventanas de lluvia intensa, anticipar tiempos muertos y reprogramar actividades críticas. Especialmente relevante para obras viales y de puentes expuestas a condiciones climáticas adversas.

**Valor para la empresa:** reducción de pérdidas por paralización de obra, mejor planificación de vaciados de concreto y movimiento de tierras.

### Pilar 2 — Seguridad con Computer Vision
Usar modelos YOLO para detectar automáticamente el uso correcto de EPPs (cascos, chalecos, arneses) en zonas de obra mediante cámaras de bajo costo. Base ya desarrollada, lista para ser formalizada como sistema.

**Valor para la empresa:** cumplimiento de la norma G.050, reducción de accidentes, evidencia documentada ante fiscalizaciones.

### Pilar 3 — Análisis geoespacial de riesgo
Usar datos del IGN, CENEPRED y SENAMHI para generar mapas de riesgo de inundación, deslizamiento y estabilidad de suelos en zonas donde se desarrollan proyectos. Directamente aplicable a estudios de prefactibilidad y diseño de rutas viales.

**Valor para la empresa:** identificación temprana de riesgos geológicos y climáticos antes de iniciar obra, insumo para expedientes técnicos.

## Hoja de ruta por etapas

### Etapa 1 — Fundamentos (meses 1-3)
- Solidificar Python: pandas, numpy, matplotlib, Jupyter
- Aprender Git y GitHub para control de versiones
- Completar la base teórica de estadística aplicada
- Primer proyecto: análisis exploratorio de datos climáticos de Lima con SENAMHI

### Etapa 2 — Aplicaciones core (meses 4-6)
- Dominar Streamlit para dashboards
- Implementar modelo de series de tiempo para pronóstico climático
- Formalizar el modelo YOLO de detección de EPPs
- Primer dashboard funcional con datos climáticos vs. cronograma

### Etapa 3 — Geoespacial (meses 7-9)
- Aprender GeoPandas y Folium
- Integrar Google Earth Engine (Python API)
- Generar primer mapa de riesgo geoespacial para zona de proyecto real
- Conectar análisis de riesgo con datos del CENEPRED y IGN

### Etapa 4 — Integración y escala (meses 10-12)
- Automatizar reportes recurrentes
- Explorar integración con BIM (datos IFC)
- Documentar casos de estudio propios
- Preparar propuesta formal de área de Data Science para constructora
